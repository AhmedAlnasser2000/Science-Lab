# V5.5e Milestone Roadmap - CodeSee Semantic Session Recording (e1-e6)

## Summary
Deliver `V5.5e` as the recording foundation milestone for later replay features:
- Record semantic runtime activity in background.
- Persist ordered runtime records plus periodic keyframes per workspace.
- Provide a minimal inspection surface in System Health (no replay controls).
- Enforce bounded retention (count + total size).
- Keep replay player/timeline controls out of scope for `V5.5f`.

## Locked Decisions
- Scope: `e1-e6` full roadmap in this slice.
- Storage baseline: `JSONL + keyframes`.
- Ordered stream contract: one append-only `records.jsonl` stream.
- Minimal review surface: `System Health -> Sessions` panel.
- Retention defaults: size + count caps with oldest-completed-first pruning.

## Out Of Scope (Explicit)
- No replay player controls (`play/pause/scrub/jump/bookmarks`) in `V5.5e`.
- No video/screen recording output.
- No cross-workspace global aggregation store.
- No broad refactor of existing CodeSee rendering behavior.

## Gate e2 - Recorder Core + Artifact Writer (`backend`)
### New/Updated Interfaces
1. `app_ui/codesee/runtime/session_recording.py` (new)
   - `SessionRecorderConfig`
   - `SessionRecorder`
     - `start_session(...) -> SessionMeta`
     - `stop_session(...) -> SessionMeta`
     - `record_event(event: CodeSeeEvent) -> None`
     - `record_state_delta(delta: dict) -> None`
     - `record_keyframe(snapshot: dict) -> None`
     - `flush() -> None`
2. `app_ui/codesee/runtime/session_schema.py` (new)
   - schema constants/version
   - validation helpers for record envelopes + keyframe envelopes
3. `app_ui/codesee/runtime/session_store.py` (new)
   - workspace path resolution
   - file append/read helpers
   - retention pruning

### Artifact Layout (Workspace-Scoped)
`data/workspaces/<workspace_id>/codesee/sessions/<session_id>/`
- `session_meta.json`
- `records.jsonl`
- `keyframes/000001.json`, `000002.json`, ...
- optional `stats.json`
- optional `session.lock` (active-session protection)

### Ordered Record Schema Contract (v1)
`records.jsonl` rows use global monotonic sequence across all record types:
- `seq`
- `type` (`event` | `delta` | `keyframe_ref`)
- `ts_utc` (ISO 8601 Z)
- `ts_ms_epoch`
- `tz_offset_minutes`
- `ts_local` (display/debug only)
- payload fields by type:
  - `event`: `kind`, `severity`, `node_ids`, `source_node_id`, `target_node_id`, `payload`
  - `delta`: `delta_type`, `node_id`, `before_ref`, `after_ref`, `metadata`
  - `keyframe_ref`: `keyframe_seq`, `filename`, `graph_state_ref`

### `session_meta.json` (v1)
- `schema_version`, `session_id`, `workspace_id`
- `started_at_utc`, `ended_at_utc`
- `started_at_local`, `ended_at_local`
- `started_at_ms_epoch`, `ended_at_ms_epoch`
- `tz_offset_minutes`
- `status` (`ACTIVE` | `COMPLETE` | `INCOMPLETE`)
- counts summary (`records`, `events`, `deltas`, `keyframes`, `corrupt_lines`)
- build/version info

### Verification
- Unit tests for schema validation + record writes.
- `python -m compileall app_ui/codesee/runtime`

## Gate e3 - Event/State Ingestion Wiring (`backend`)
### Integration Points
- Wire recorder into existing CodeSee runtime event path (screen/runtime hub bridge) without changing current monitor semantics.
- Record:
  - bus request/reply events
  - span start/update/end derived state deltas
  - monitor state transitions
  - active trace changes (`follow/pin/unpin`)

### Rules
- No semantic timer decay.
- Recorder must be non-blocking and fail-soft (recording errors cannot break live UI).

### Verification
- Deterministic tests with synthetic event streams.
- Assert expected record counts and strictly increasing `seq`.

---

## Gate e4 - Keyframe Cadence + Reconstructability (`backend`)
### Deliverables
- Keyframe policy in recorder config:
  - every `K` records (default fixed cadence)
  - forced keyframe on session stop
- Reconstruction helper:
  - load nearest keyframe + apply subsequent `records.jsonl` entries to recover terminal monitor/trace state.

### Rules
- Cadence meaning is structural (count-based), not activity-window semantics.
- Reconstructability must be deterministic.

### Verification
- Test: generated session reconstructs same terminal monitor/trace state.
- Test: missing/corrupt keyframe handled gracefully with fallback path.

## Gate e6 - Retention + Hardening + Milestone Verification (`backend`)
### Deliverables
- Retention defaults:
  - `max_sessions_per_workspace = 20`
  - `max_total_mb_per_workspace = 1024`
- Prune policy:
  - remove oldest completed sessions first
  - never delete active session
- Active-session protection:
  - write `session.lock` with `pid` + `started_at_ms_epoch` (best effort)
  - treat live recorder handle as active in-process authority
- Failure-mode handling:
  - skip malformed JSONL lines
  - increment `corrupt_lines`
  - surface `corrupt_lines=N` in System Health
  - if `session_meta.json` is missing/partial, mark as `INCOMPLETE` and keep listable

### Verification
- Unit tests:
  - prune by count
  - prune by size
  - mixed cap conflict behavior
  - active-session protection
  - malformed-line tolerance + `corrupt_lines` propagation
  - `INCOMPLETE` session listing behavior
- Final suite:
  - targeted `pytest` for recording modules
  - compile checks for touched packages

## Test Cases And Scenarios
1. Session lifecycle:
   - start -> ingest records -> stop writes complete metadata and counts.
2. Keyframe integrity:
   - cadence creates expected keyframes and reconstructs terminal state.
3. Ordering guarantees:
   - `seq` strictly monotonic in `records.jsonl`.
4. Fault tolerance:
   - malformed lines skipped and counted without app crash.
5. Retention:
   - count/size pruning removes oldest completed sessions only.
6. UI inspection:
   - System Health Sessions panel lists sessions and shows details without replay UI.
7. Compatibility:
   - loader tolerates unknown future fields (forward-safe parsing).

---

## Assumptions And Defaults
- `V5.5e` is recording-first infrastructure, not replay UX.
- Existing monitor/trace runtime events are sufficient for initial capture.
- JSONL is preferred for append performance and partial recovery.
- UTC/epoch is canonical for storage; local time remains for display/debug.
- Workspace pathing uses existing project/workspace resolution utilities.

---

## Milestone Acceptance (V5.5e)
`V5.5e` is complete when:
1. Semantic sessions are recorded in background to workspace-scoped artifacts.
2. Record/keyframe schema contracts are stable and versioned.
3. Sessions are reconstructable for later replay consumption.
4. System Health Sessions panel can inspect recorded sessions minimally.
5. Retention/pruning prevents unbounded growth.
6. Replay player remains deferred to `V5.5f` with no scope leakage.
