# V5.5f Milestone Roadmap - CodeSee Session Replay Player (f1-f7)

## Summary
Deliver `V5.5f` as the replay player milestone on top of the `V5.5e` recording baseline.

This milestone must consume existing session artifacts as-is:
- `records.jsonl` (single ordered stream)
- `keyframes/*.json`
- `session_meta.json`

Primary outcomes:
- Deterministic seq-driven replay timeline.
- CodeSee-first replay controls (`play/pause/speed/scrub/jump`).
- Marker + bookmark support with per-session sidecar persistence.
- Replay-exclusive rendering mode with explicit live buffering behavior.

No video export is introduced in `V5.5f`.

## Locked Decisions
- Timeline engine is seq-driven.
- Jump by seconds maps to nearest record using `ts_ms_epoch`.
- Replay state scope is monitor+trace first.
- Replay controls live in CodeSee primary UI.
- Bookmarks ship in `V5.5f`.
- Bookmarks persist in `<session>/bookmarks.json`.
- Replay mode is replay-exclusive (live overlay rendering suppressed while active).
- Live buffering rule: while replay is active, live runtime events may continue to be collected, but must not mutate replay state or render until replay exits.
- Bookmark persistence helpers are planned under `app_ui/codesee/storage/session_store.py`; replay engine stays in `app_ui/codesee/runtime/`.

## Out Of Scope (Explicit)
- Pixel/video replay export.
- Cross-workspace/global replay aggregation.
- Full visual parity (`micro-animations` / exact pulse cosmetics) with live mode.
- Full event/badge/pulse cosmetic reenactment.
- `V5.5g` user interaction timeline capture.

Note: monitor+trace semantics are in scope (`status/errors/spans/basic activity semantics`), while cosmetic parity is out of scope.

## Gate f3 - Timeline Index + Keyframe Seek Engine (`backend`)
### Deliverables
- Implement timeline index by:
  - `seq -> record`
  - `ts_ms_epoch -> nearest seq`
- Implement seek API:
  - `seek_to_seq(...)`
- Seek algorithm must use nearest prior keyframe and apply deltas forward to target seq.
- Corrupt/missing keyframe fallback must be deterministic and warning-backed.

### Verification
- Extend `tests/test_codesee_session_replay.py` for keyframe-assisted seeking.
- Add corrupt/missing keyframe and malformed JSONL tolerance cases.

## Gate f4 - Playback Controller (`backend`)
### Deliverables
- Implement `ReplayController` in `session_replay.py`:

  - scrub to seq/time
  - jump `+/-N` seconds
- Default jump step:
  - `N = 5s`
- Speed presets:
  - `0.25x, 0.5x, 1x, 1.5x, 2x, 4x`
- Playback tick behavior must be deterministic for identical inputs.

### Verification
- `tests/test_codesee_session_replay.py` includes replay tick determinism and jump behavior coverage.

## Assumptions And Defaults
- Jump step `N = 5s`.
- Speed presets: `0.25x, 0.5x, 1x, 1.5x, 2x, 4x`.
- Seek uses nearest prior keyframe then applies deltas to target `seq`.
- Unknown future fields in records/meta/bookmarks are ignored (forward-safe parsing).
- Replay does not modify session records/keyframes/meta; only bookmark sidecar is mutable.

---

## Milestone Acceptance (V5.5f)
`V5.5f` is complete when:
1. Replay session selection and deterministic scrubbing are available in CodeSee.
2. `play/pause/speed/jump` controls function correctly.
3. Keyframe-assisted seek is correct and resilient to corruption/missing artifacts.
4. Bookmarks can be created, listed, jumped, removed, and persisted.
5. Exiting replay restores normal live monitor/trace rendering behavior.
