# V5.5c Lens Palette — “Box” Design Reference (for Codex prompts)

Use this as the UI target for all **V5.5c** follow-up work that touches the Lens Palette / lens selection UI.

## High-level intent
- A **compact, floating “box” palette** (not the radial wheel) that feels like a modern launcher:
  - Title + close button
  - Search field
  - Grid of icon tiles (primary entry points)
  - Footer actions (“Recent”, “More…”)

## Layout spec
- Container:
  - Floating panel (QDialog/QFrame) with rounded corners
  - Soft shadow + subtle border
  - Semi-opaque background (blur/glass if available; otherwise translucent solid)
- Header row:
  - Left: **“Lenses”** title
  - Right: **✕** close button (icon-only, hover highlight)
- Search:
  - Full-width rounded search input with leading magnifier icon
  - Placeholder: “Search”
- Tile grid:
  - 3 columns × N rows (wrap as needed)
  - Each tile:
    - Top: line icon (centered)
    - Bottom: label text (centered)
    - Hover: slight lift / brighter border
    - Selected: accent outline + slightly brighter icon/text
- Footer:
  - Left: **Recent**
  - Right: **More… >** (or “More…” with chevron)

## Icon style requirements (match the reference screenshot)
- Use the **existing CodeSee icon set/style** already used in the app:
  - Thin/clean **line icons**, monochrome with the app accent tint
  - Consistent stroke weight and visual rhythm across tiles
- Size:
  - Tile icon: ~24–28px glyph area (or 32px widget if padding is used)
- Color:
  - Default: slightly muted foreground
  - Hover/selected: accent tint (use theme/accent from QSS or palette highlight)
- Do **not** introduce mixed icon families (no filled icons next to line icons).

## QSS / styling guidance (Qt)
- Prefer tokens already present in your QSS pack (quantum_smooth):
  - Background: panel bg (semi-opaque)
  - Border: subtle 1px
  - Radius: 14–18px
  - Shadow: QGraphicsDropShadowEffect (soft, low alpha)
- Tile widgets:
  - Rounded radius ~12px
  - Padding: 10–12px
  - Spacing between tiles: 10–14px
- Text:
  - Title: medium weight, clear
  - Labels: slightly smaller, muted

## Interaction requirements
- Clicking a tile **must** apply the selected lens immediately:
  1) Update active lens id in controller/state
  2) Update palette highlight/checked tile
  3) Trigger CodeSee view refresh/re-render
- Palette may close after selection **but selection must apply** (close on next tick is OK).
- Keyboard:
  - Esc closes
  - Search focuses on open (optional but preferred)
  - Arrow keys navigate tiles (optional)
